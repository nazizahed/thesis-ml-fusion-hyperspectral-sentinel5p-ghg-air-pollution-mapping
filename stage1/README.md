# Stage 1: Cross-sensor Visibility

This stage analyzes whether high-resolution Carbon Mapper/Tanager methane
plumes and coarse-resolution Sentinel-5P methane observations are visible in
the same spatial and temporal context.

The workflow is intended for cross-sensor contextual comparison. It does not
claim that Sentinel-5P validates or resolves an individual Tanager plume.

## Main Notebook

```text
stage1_cross_sensor_visibility_final.ipynb
```

## Workflow

1. Load a Carbon Mapper plume CSV generated by Stage 0 or obtained from the
   Carbon Mapper API.
2. Normalize gas labels and retain valid Tanager CH4 plume records.
3. Read each plume visualization and concentration GeoTIFF.
4. Calculate plume-mask statistics: `n_valid`, `mean_ch4`, `min_ch4`, and
   `max_ch4`.
5. Min-max normalize `n_valid` and `mean_ch4`.
6. Rank plumes using an equally weighted normalized score.
7. Select the top 20 plumes as seed events.
8. Associate lower-ranked plumes within 50 km and +/-120 hours.
9. Query daily Sentinel-5P XCH4 context from five days before to five days
   after each seed event.
10. Export tables, figures, Folium maps, and an interactive geemap view.

See [../docs/stage1-technical-reference.md](../docs/stage1-technical-reference.md)
for equations, algorithm details, spatial assumptions, and limitations.

## Requirements

- Python 3.10-3.12 is recommended.
- The packages listed in the repository-level `requirements.txt`.
- A Stage 0/Carbon Mapper plume CSV.
- Internet access to Carbon Mapper raster URLs.
- A Google account with Earth Engine access.
- A Google Cloud project registered for Earth Engine use.

Install the shared environment from the repository root:

```bash
python -m pip install -r requirements.txt
```

## Required Input

For the complete workflow, the plume CSV should contain:

```text
plume_latitude
plume_longitude
datetime
gas
platform
plume_tif
con_tif
```

Useful optional fields include:

```text
plume_id
emission_auto
ipcc_sector
instrument
source_name
```

If `plume_id` is absent, the notebook creates deterministic identifiers from
the CSV row order. If `platform` is present, only rows labeled `TANAGER` are
retained. Missing or inaccessible raster URLs prevent mask statistics from
being calculated for the affected rows.

## Configuration

Open the notebook and edit the configuration cell before running the
workflow.

### Input and output paths

The supplied defaults are Colab paths:

```python
RAW_CM_CSV = Path("/content/plumes_example.csv")
OUT_DIR = Path("/content/stage1_outputs")
```

For a local checkout, use paths such as:

```python
RAW_CM_CSV = Path("path/to/plumes.csv")
OUT_DIR = Path("stage1_outputs")
```

`stage1_outputs/` is ignored by Git because it can contain generated tables,
figures, and maps.

### Main parameters

| Parameter | Default | Meaning |
| --- | ---: | --- |
| `TOP_N` | 20 | Number of top-ranked seed plumes. |
| `W_NVALID` | 0.5 | Weight assigned to normalized valid-pixel count. |
| `W_MEANCH4` | 0.5 | Weight assigned to normalized mean CH4. |
| `SPATIAL_RADIUS_KM` | 50 | Maximum association distance from a seed. |
| `TEMPORAL_WINDOW_HOURS` | 120 | Maximum absolute time difference. |
| `S5P_BUFFER_M` | 50,000 | Earth Engine reduction radius around a seed. |
| `S5P_DAYS_BEFORE_AFTER` | 5 | Daily S5P offsets on either side of `t0`. |
| `S5P_SCALE_M` | 7,000 | Reduction scale for the S5P L3 product. |
| `TARGET_RANK` | 4 | Seed rank used for detailed maps and plots. |

### Earth Engine project

Change `EE_PROJECT` to a Google Cloud project for which your account has
Earth Engine access:

```python
EE_PROJECT = "your-earth-engine-project-id"
```

The project identifier in the notebook is an example from the development
environment and should not be assumed to work for other users.

## Interpretation

- `n_valid` is the number of valid concentration pixels on the plume raster
  grid, not a physical plume area.
- `mean_ch4` is the mean raster value over valid plume pixels. Its meaning and
  units follow the source Carbon Mapper product.
- The weighted score is a relative ranking within the current input dataset.
- A lower-ranked plume is assigned to at most one seed, prioritizing the
  smallest absolute time difference and then the smallest distance.
- The Tanager proxy `mean_ch4 * n_valid` is a visualization proxy, not a
  formal emission-rate estimate.
- Sentinel-5P results describe daily coarse-scale XCH4 context in a 50 km
  buffer. They are not direct confirmation of an individual plume.

## Troubleshooting

**`RAW_CM_CSV not found`**

Update `RAW_CM_CSV` to the actual Stage 0 CSV path.

**No Tanager CH4 rows**

Inspect the `gas`, `platform`, coordinate, and datetime values in the input
CSV. The platform comparison is case-insensitive but expects `TANAGER`.

**Raster statistics are null or zero**

Verify that `plume_tif` and `con_tif` contain accessible HTTP(S) or local
GeoTIFF paths. Network, GDAL, CRS, or source-raster issues are intentionally
converted to null statistics so one failed raster does not stop the run.

**Earth Engine initialization fails**

Confirm that `EE_PROJECT` exists, Earth Engine is enabled for it, and your
Google account has permission to use it.

**No S5P value on a date**

The selected daily collection can be empty or fully masked over the buffer.
Null daily values are expected in some locations and periods.

**Target rank map fails**

Set `TARGET_RANK` to an available rank between 1 and `TOP_N`. The selected
seed must also exist after filtering and ranking.
